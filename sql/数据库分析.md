
# 课程表、文章表的联系
## 平行独立的主体
课程和文章应该是两个平行的主体，原因是：
 1. 不同的内容形态
- 课程：系统化的学习内容，包含多个章节、视频/音频资源等
- 文章：独立的知识点、经验分享、资讯等文本内容
2. 不同的使用场景
- 课程：适合系统化学习，需要按步骤学习
- 文章：适合碎片化阅读，可以随时查看

### 6. 展示效果
1. 课程页面：
- 显示课程基本信息
- 关联的教案和补充资料
- 推荐相关的独立文章

2. 知识库页面：
- 显示所有独立文章
- 可以按分类、标签筛选
- 支持难度等级筛选


# 课程表、课程章节表、课程资源表的联系
## 1. 表之间的关系图
study_lesson (课程表)
    │
    ├──── study_lesson_chapter (课程章节表)
            │
            └──── study_lesson_resource (课程资源表)
    
## 2. 具体关联关系
1.  课程 -> 章节：一对多关系
    -- study_lesson_chapter表中的关联字段
    `lesson_id` bigint(20) NOT NULL COMMENT '课程ID'
    - 一个课程可以包含多个章节
    - 每个章节必须属于一个课程
    - 通过lesson_id进行关联
- 
2. 章节 -> 资源：一对多关系
-- study_lesson_resource表中的关联字段
`lesson_id` bigint(20) NOT NULL COMMENT '课程ID',
`chapter_id` bigint(20) DEFAULT NULL COMMENT '章节ID'
- 一个章节可以包含多个资源
- 资源必须属于一个课程，可以选择性属于某个章节
- 同时保存lesson_id和chapter_id便于查询和管理

## 3. 业务场景示例
- 创建新课程：
    1. 创建课程基本信息
       └── INSERT INTO study_lesson
    2. 创建课程章节
       └── INSERT INTO study_lesson_chapter
    3. 上传课程资源
       └── INSERT INTO study_lesson_resource

- 学习课程内容：
```
    课程页面
    ├── 课程信息 (study_lesson)
    │   ├── 课程标题
    │   ├── 课程描述
    │   └── 课程标签
    │
    ├── 章节列表 (study_lesson_chapter)
    │   ├── 章节1
    │   │   ├── 视频1 (study_lesson_resource)
    │   │   ├── 音频1 (study_lesson_resource)
    │   │  
    │   │
    │   └── 章节2
    │       ├── 视频2 (study_lesson_resource)
          
    
   
    
```

## 4. 数据完整性维护

- 删除课程时：
    -- 需要级联删除或处理：
    1. 相关章节 (study_lesson_chapter)
    2. 章节资源 (study_lesson_resource)
    
- 删除章节时：
    -- 需要处理：
    1. 章节下的资源 (study_lesson_resource)
    2. 更新其他章节的排序
    

这种设计的优点：
- 结构清晰，层次分明
- 支持灵活的课程内容组织
- 便于内容管理和展示
